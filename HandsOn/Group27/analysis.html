URI Strategy


Content Negotiation:

The application will handle content negotiation with ease, using "/" for resources and "#" for classes/properties, in case of a request the response will redirect to a related page or by default will download the ontology.

Pattern URI for resources:
http://www.semanticweb_g27.org/resource/{Class_name}/{resource_id}

- The id will be the identifier provided by the dataset followed by a capital letter.

- In case that the dataset doesn't provide an identifier, a random 8-digit number will be applied instead.

Pattern URI for Classes and Properties:

http://www.semanticweb_g27.org/ontology/EnergyCons#[{Class_name} | {Property_name}]

- The name for the ontology is "EnergyCons".

- The class name starts with a capital letter.

- Properties name starts with lowercase.
